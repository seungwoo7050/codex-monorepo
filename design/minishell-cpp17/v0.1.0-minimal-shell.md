# minishell-cpp17 v0.1.0 - 최소 명령 실행 설계

## 개요
- 한 줄 입력을 받아 공백 단위로 파싱하고 단일 명령을 실행한다.
- POSIX fork/execvp 조합으로 자식 프로세스를 생성하며 종료 코드를 출력한다.

## 목표 및 범위
- 입력: 표준 입력에서 한 줄만 읽는다.
- 파싱: 공백 기준 분리, 따옴표나 특수 문법은 지원하지 않는다.
- 실행: execvp로 PATH 기반 프로그램을 실행한다.
- 출력: 명령 결과와 종료 상태를 표준 출력으로 전달한다.

## 외부 동작
- 프롬프트("$ ")를 출력하고 입력을 기다린다.
- 예) `echo hello` 입력 시 `hello`와 `exit status: 0`을 순서대로 출력한다.
- 잘못된 명령 실행 시 오류 메시지와 비정상 종료 코드를 출력한다.

## 내부 설계
- `splitArguments`: std::istringstream로 공백 분리.
- `executeCommand`: fork 후 자식에서 execvp 호출, 실패 시 127 종료.
- 부모는 waitpid로 상태를 받아 정상/시그널 종료를 판별한다.

## 테스트 전략
- tests/run_echo.sh에서 `echo test-message` 실행 결과와 종료 코드 확인.
- 오류 처리 확인을 위해 execvp 실패 시 오류 메시지가 표시되는지 수동 점검 가능.

## 빌드/실행 참고
- CMake 3.16+ 필요, `cmake -S . -B build && cmake --build build`.
- 실행 예: `echo "echo hi" | ./build/minishell`.
